Aspectos técnicos

Data Binding

Las plantillas de angular se compilan para producir vistas, luego cualquier cambio en ellas se refleja en el modelo, y cualquier cambio en el modelo se propaga a la vista, que pasa a ser una proyección instantánea de este.

En la mayoría de los sistemas de plantillas, se mezclan los datos en una sola dirección: se mezclan las plantillas y los modelos en una vista. Luego, los cambios de los modelos no se reflejan automáticamente en la vista, y cualquier cambio que el usuario relaiza en la vista no se ve automáticamente reflejado en el modelo.

El Data-binding en AngularJs es la sincronización automática de los datos entre el modelo y la vista. La forma en que se implementa deja al modelo ser la "single-source-of-truth" de la aplicación.

Modulos

¿Qué es un módulo?

Contenedores para diferentes partes de la aplicación (controladores, servicios, entre otros).

AngularJs no tiene un método principal. Los módulos se encargan de especificar el arranque y la secuencia de una aplicación.

Esto permite código fácilmente reusable y simple de entender.

Scopes

¿Qué es un scope?

Es el objeto que refiere al modelo de la aplicación. Provee de contexto para la ejecución de expresiones. Son organizados de forma jerárgica. Son el "pegamento" entre el controlador y la vista.

Cuando AngularJs empieza a ejecutarse, compila el código de las plantillas HTML para generar las vistas y empieza a realizar los enlaces entre la aplicación y el $rootscope, que pasa a ser el padre de todos los objetos $scope que se instancien, siendo el objeto más cercano a el contexto global en una aplicación AngularJs.

No es una buena práctica desarrollar demasiado código en el $rootScope, ya que puede llevar a colisiones, y se pierde el control del contexto en la aplicación

Descripción de un scope:

  • Observan las modificaciones en el modelo ($watch)
  • Propagan estos cambios hasta las vistas ($apply)
  • Pueden anidarse como hijos o aislados para controlar el acceso a las propiedades de los componentes de las aplicaciones

Routing

¿Cómo se manejan las rutas en AngularJs?

Las rutas de las aplicaciones en AngularJs se manejan a través de el $routeProvider que es el proveedor del servidor de rutas $route que requiere el módulo ngRoute.

Controladores

En AngularJs, los controladores son funciones constructoras de JavaScript que se utilizan para aumentar al Scope.

Cuando se instancian, se hace disponible un nuevo Scope hijo para injectarse como parametro en la función constructora del controlador (parámetro $scope).

Los controladores se usa para:

  • Instanciar e inicializar el objeto $scope dentro de su contexto.
  • Manejar el comportamiento del scope.

Muchas veces se comete el error de utilizar los controladores para otro tipo de tareas. Desde un controlador no se debe:

  • Manejar el DOM. Para ello AngularJs provee del databinding y directivas.
  • Manejar entradas de datos. Los controles de formularios de Angulajs están para esto.
  • Hacer filtros para salidas. Los filtros hacen esta tarea.
  • Compartir código entre controladores. Los servicios están para ello.
  • Controlar el ciclo de vida de otros componentes.

Sintaxis para los controladores: 

.controller('Home', ['$scope','$rootScope', function($scope,$rootScope) {}]);

Directivas

¿Qué son las directivas?

Son marcadores en elementos del DOM Que le dicen al compilador HTML ($compile) de AngularJs que ese elemento en cuestión tendra asignado un comportamiento específico.

AngularJs tiene un amplio set de directivas incluidas, como ng-repeat que son de extrema utilidad para cualquier desarrollador que se aventure con este framework, ademas de ofrecer la posibilidad de crear nuestras propias directivas a nuestro antojo.

Ejemplo de la sintaxis para la directiva de un controlador: 

<div ng-controller="Controller"></div>

Servicios

Los servicios se utilizan en la aplicación para compartir código a través de ella.

Los controladores son instanciados sólo cuándo se necesitan y son descartados cuando se sale de su contexto. Cada vez que cambiamos de ruta o se carga una vista, el controlador actual se limpia.

Los servicios son objetos que se instanacian una sola vez por aplicación, y se cargan solo cuándo son necesarios (lazy-load).

Registrar y usar un servicio

Los servicios se registran en los módulos a través de diferentes métodos.

Para usar un servicio se debe agregar la dependencia al componente donde se quiere usar, de esta manera se injecta y se vuelve disponible.

Filtros

Los filtros proveen de una manera para dar formato a los datos de salida.

Para utilizar los filtros en nuestra aplicación, se usa el símbolo | en la plantilla HTML, o se usa el servicio $filter desde el lado de JavaScript.

Para usarlos en expresiones en una vista se usa de la siguiente manera: 

{{ expresión | filtro }}

Los filtros pueden ser anidados usando varios |. 

{{ expresión | filtro1 | filtro2 | ... }}

También pueden tener argumentos: 

{{ expresión | filtro:argumento1:argumento2:... }}

Eventos

Son información que se emite y propaga a través de la aplicación que contienen información acerca de algo que sucede en ella.

Así como el navegador responde a eventos como el click del mouse o el scroll, AngularJs responde a eventos propios del marco de desarrollo. Esto nos provee de la ventaja de una mayor comunicación entre componentes bastante separados.

Los eventos viajan a través de el arbol de herencia de los scopes.

Para enviar un evento hacia arriba desde algún scope hijo se usa la función $emit(). 

$scope.$emit('evento_disparado',$scope.info);

Para enviar un evento hacia abajo en el arbol de herencia, se usa $broadcast(). 

$scope.$broadcast('evento_disparado',$scope.info);

Para recibir los eventos, se utiliza la siguiente sintaxis: 

$rootScope.$on('$routeChangeSuccess', function(){});

Al recibir un evento, si se quiere detener su propagación por el resto de los scopes, se usa el método $stopPropagation(), de esta manera se puede llegar a tener un mejor control sobre los eventos y evitar posibles colisiones entre scopes. 

$scope.$on('$includeContentLoaded',function(event){event.stopPropagation();});
